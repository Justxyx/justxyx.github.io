这一章没什么好聊的， DHCP 用的少，后续再补充。主要关注 二层转发 & 三层转发即可。

| 特性 / 维度     | 二层广播 (L2 Broadcast)                            | 三层广播 (L3 Broadcast)                                                 |
| ----------- | ---------------------------------------------- | ------------------------------------------------------------------- |
| **主要协议层**   | 数据链路层 (Layer 2)                                | 网络层 (Layer 3)                                                       |
| **目的地址**    | MAC 广播地址 `FF:FF:FF:FF:FF:FF`                   | IP 广播地址，例如 `192.168.1.255`（定向广播）、`255.255.255.255`（局域网广播）           |
| **转发方式**    | 交换机在同一 VLAN 内泛洪到所有端口                           | 默认：路由器不转发广播 → 只在本地子网有效；<br>定向广播：管理员允许时，路由器将封装为 L2 广播发送到目标子网         |
| **处理流程**    | 帧到达交换机 → 查 MAC 表 → 未知 MAC 或广播 → 转发到 VLAN 内所有端口 | 数据报到达路由器 → 查路由表 → <br>默认：丢弃广播；<br>定向广播：匹配目标网络 → 封装为 L2 广播 → 发送到目标子网 |
| **作用 / 目的** | 发现未知 MAC（如 ARP 请求）、局域网内通知                      | 局域网内发送消息给所有主机（如 DHCP Discover、NetBIOS 广播）                           |
| **设备影响**    | 交换机只在 VLAN 内泛洪，广播风暴可影响整个 VLAN                  | 默认只影响本地子网；定向广播跨子网需路由器允许，否则不转发                                       |
| **常见协议举例**  | ARP 请求 (`Who has IP?`)、Spanning Tree BPDU 广播   | DHCP Discover、NetBIOS 广播、Wake-on-LAN 广播                             |
| **限制**      | 不能跨网段                                          | 默认不跨路由器；跨网段广播需开启定向广播；滥用可能造成广播风暴                                     |
| **广播域范围**   | VLAN 内所有端口                                     | L3 子网内所有主机；跨子网需定向广播                                                 |
